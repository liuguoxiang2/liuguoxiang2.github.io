linux用户和组的管理

一.概念及目的

  • Authentication:认证
  • Authorization:授权
  • Accounting:审计

Linux 用户：username/uid
管理员：root,0
普通用户：1-65535
系统用户：1-999
对守护程序获取资源进行权限分配
登录用户：（uid）1000+（centos7）
交互式登录
Linux组
管理员组：root,0
普通组，1000+
Linux安全上下文：
运行中的程序
以进程发起者的身份运行：
root:cat
进程所能够访问所有资源的权限取决于进程的发起者的身份

Linux 组的类别：
用户的基本组（主组）
组名同用户名：且仅包含一个用户，私有组
用户的额外组

Linux用户和组相关的配置文件：
/etc/password:用户及属性的信息
/etc/group：组极其属性信息
/etc/shadow:用户密码以及相关属性

二.有关配置文件的详解

1./etc/passwd
user2:x:1001:1001::/home/user2:/bin/bash
(1).用户名
(2).密码（这里由于为了安全性的问题，都用x代替，真正的密码都在/etc/shadow）
(3)用户标识号，即uid，
(4)组标号
(5)用户详细信息，备注信息
(6)用户主目录，如果没有在创建指定，即为/home/username
2./etc/shadow
以user2:!!:18115:0:99999:7:::为例
(1).用户名
(2).加密后的密码，如果这一栏!或者*的话，说明这是一个不能登录的账户。
(3).最近改动密码的日期（从1970年1月1日算起，把这个当作unix的元年
(4).最小密码使用日期：即从上次更改密码算起，最少隔多少天才能修改修改密码
(5).最大密码使用日期：从上次更改密码，隔多少天必须修改密码若99999为没有限制
(6).密码过期预警时间：最大密码使用日期到期之前的天数，超过这个天数还没有修改，系统会提示修改密码
(7).宽限期：超过最大密码使用日期之后，如果仍然没有修改，则还会有几天的宽限日期
(8)账号失效日期：超过这个日期还没有修改密码，系统会将这个用户锁掉，此时只能找管理员。
(9)保留项

用户信息的增删改查
（1）useradd命令
常用选项：
-c 加上备注文字，保存在passwd的备注栏中
-d 指定用户的家目录
-g：指定用户所属的群组
-G:指定用户所属的附加群组
-r:建立系统账号
-s:指定用户登入后所使用的shell
-u:指定用户的ID

（2）查看用户的id信息
id [options].. [user]
-u:uid
-g:gid
-g:groups
-na：Name

切换用户或者以其他用户的身份执行命令
切换用户的方式：
su UserName:非登陆式切换
su - uesrname

用户属性的修改:
usermod -d 修改家目录 :
tips:单原有的家目录文件不会复制到新家目录，若要移动使用-m

（2）.用户的增加

给用户添加密码（只有管理员又权限）
passwd 【options】 usrname:修改指定用户的密码，仅root用户权限
passwd :修改自己的密码

常用选项
-l:锁定指定用户
-u:解锁指定用户
-n minday:指定密码最小使用期限
-x maxdey:指定密码最大使用期限
-w:提前多少天开始警告
-i:

–stdin:从标准输入接受用户名
echo “password”|passwd –stdin username

删除用户
userdel [option].. login
-r :删除用户的家目录

组属性修改
groupmod
常用属性:

  • -n:新名字
  • -g:新的gid

组删除：groupdel
groupdel group

newgrp:临时切换基本组

修改用户密码属性
change
-d last day
-E 密码到期的时间
-m 密码最小的使用时间
-M 密码最大的有效时间
-W
-I 宽限期

权限管理

对于文件来说
r:可以使用cat等文件查看工具获取其内容
w:可以修改其内容
x:可以把此文件提请内核启动为一个进程

目录：
r::可以用ls查看此目录中文件列表
w:可在此目录中修改或删除文件
x:可以使用ls -l查看此目录中文件，可以cd进入目录里
